## 1. 한 줄 컨셉

**5분짜리 역방향 보스전 로그라이크.** 플레이어는 던전 주인. 모험가 영웅 한 명이 자동으로 던전을 돌파해오고, 플레이어가 배치한 몬스터 무리가 자동 전투. 영웅 HP 10%마다 "몬스터 강화/교체" 선택지(패시브), 30초마다 "영웅 저주 / 몬스터 버프" 카드(액티브)가 뜬다. 5분 안에 영웅을 죽이면 승리.

## 2. 장르 / 플랫폼 / 스택

- **장르**: 자동전투 로그라이크 (Vampire Survivors-like) + 덱빌딩 + 역방향 던전 운영
- **시점**: 2.5D 탑다운
- **엔진**: Unity
- **타깃 플랫폼**: PC / 모바일 (둘 다 고려, 우선순위 TBD)
- **플레이 형식**: 싱글 플레이
- **서버**: 데이터 저장 용도 (세이브, 메타 진행, 리더보드)

## 4. 코어 루프 (한 런 = 5분)

### 4.1 런 시작 — 지속 스폰 모델
- 영웅 한 명이 던전 정중앙에서 등장, 자동으로 전투 시작.
- 던전에는 영웅을 중심으로 한 **원(ring) 둘레에 Spawner 들이 배치**되어 있다. 각 Spawner 는 한 판 내내 **고정 주기**로 지정 몬스터를 한 마리씩 흘려보낸다 — 시작 시 1회 배치하는 모델이 아니라, 5분 동안 끊임없이 몬스터가 보충되는 **지속 스폰(Continuous Spawn)** 모델이다.
- 몬스터는 ring 둘레에서 태어나 안쪽(영웅 방향)으로 수렴한다 → 영웅을 사방에서 포위 압박. 영웅은 5분간 끊임없이 수렴해오는 몬스터 무리와 교전한다.
- 필드에 동시에 존재하는 몬스터 총량은 **글로벌 캡**으로 제한된다 — 캡에 도달하면 Spawner 가 자연 백오프하여, 영웅 주변이 늘 적당히 둘러싸이되 화면이 터지지 않는다.
- 카드는 이 지속 스폰 구조 위에서 의미를 갖는다 — 강화는 종(種)의 영구 글로벌 버프, 추가는 Spawner 출력 증가, 교체는 Spawner 출력 종 변경. 즉 카드 픽은 던전의 생산 라인을 영구히 바꾸는 결정이다.

> Spawner 개수·반지름·스폰 주기·캡 수치 등 구현 디테일은 `docs/design/continuous-spawn-round.md` 가 단일 진실. 본 절은 모델 개념만 기술한다.

### 4.2 진행 중 (0:00 ~ 5:00)

두 종류의 이벤트가 독립적으로 발생:

**[패시브 이벤트] HP 10%마다**
- HP 90% → 선택지 (3택 1)
- HP 80% → 선택지
- ... (HP 10%까지)
- **최대 9번**
- 종류: 강화 / 교체 / 추가 / 진화 / 환경(독안개 등)

**[액티브 이벤트] 30초마다**
- 0:30, 1:00, 1:30, ..., 4:30
- **최대 9번**
- 종류: 저주(영웅 디버프) / 버프(몬스터 강화) / 와일드
- 매번 3장 풀 (저주 1 + 버프 1 + 와일드 1) 중 선택

**총 의사결정 횟수: 최대 18회 (평균 약 17초마다 1회)**

### 4.3 겹침 처리
- **일시정지 + 큐 방식**
- 패시브와 액티브가 동시에 트리거되면, 게임 일시정지 후 한 선택씩 또렷하게 순차 처리
- 우선순위: 패시브 먼저 → 액티브

### 4.4 종료 조건
- **승리**: 영웅 HP 0
- **패배**: 5:00 도달 (타임오버 = 던전 함락)

## 5. 빌드 시스템 — 몬스터

### 5.1 카테고리 (예시, 풀 디자인은 TBD)

- **탱커**: 위스프, 레이스 — HP↑, 영웅 진로 방해

- **광역**: 위스프 킹(분열), 좀비(자폭)
- **보조**: 네크로맨서(부활), 샤먼(버프)

### 5.2 시너지 방향성 — 4축 × 3Tier (v0.6)

카드 리뉴얼 v0.6 (2026-05-31) 부터 시너지는 **4개 빌드 축(Tank/Dps/Debuff/Swarm)** 위에 정의된다. 같은 축 카드를 N장 픽하면 **3장 / 5장 / 7장** 임계에서 즉시 1회 발화 (Tier 1·2·3). 같은 카드 K번 중첩 픽도 빌드 카운트에 그대로 누적된다.

| 축 | Tier1 (3장) | Tier2 (5장) | Tier3 (7장) |
|---|---|---|---|
| **Tank** | Wisp·Wraith HP ×1.3 (글로벌 영구) | Wisp·Wraith Power ×1.2 (글로벌 영구) | 필드 글로벌 캡 +6 (18→24, 영구) |
| **Dps** | Reaper·Hex Power ×1.3 | Reaper·Hex Cooldown ×0.8 (=공속 +25%) | Reaper·Hex Range ×1.3 |
| **Debuff** | Plague SlowFactor ×0.8 (강한 둔화 추가) | HeroAttackDown 자동 등록 (영웅 공격력 ×0.85 영구, 카드와 곱연산 누적) | 영구 출혈 등록 — 영웅 이동 시 1s당 HP -1%, 라운드 끝까지 |
| **Swarm** | Phantom·Wisp MoveSpeed ×1.3 | 모든 스포너 주기 ×0.85 (영구) | 모든 스포너 동시 출력 +1 (영구) |

수치·발동 규칙·누적 정책은 `docs/design/card-renewal.md` §4 (12개 빌드 시너지 Tier 표) 가 단일 진실이다. 시너지가 *선택지 카드에 자연스럽게 노출되도록* 카드 픽 팝업 상단에 4축 빌드 카운트 바를 노출한다 (`card-renewal.md` §8).

## 6. 액티브 카드 예시 (참고용, 풀 디자인은 TBD)

**저주 (영웅 디버프)**
- 공포: 영웅 5초간 도망
- 출혈: 영웅 이동 시 HP -1%
- 시야 안개: 다음 방 안 보임
- 저주받은 검: 영웅 데미지 -50%

**버프 (몬스터 강화)**
- 광폭화: 모든 몬스터 공격속도 +50% (10초)
- 증식: 위스프/팬텀류 즉시 2배 소환
- 피의 갈증: 처치 시 주변 몬스터 HP 회복
- 암흑의 가호: 1분간 받는 데미지 -30%

## 7. 메타 진행 (런 사이 영구 성장)

5분 런이라 메타 진행이 필수.

- **던전 영주 레벨**: 런마다 경험치 누적, 새 몬스터/카드 해금
- **영혼석(소울)**: 런 결과에 따라 획득, 패시브 업그레이드 상점에서 사용
- **새 영웅 해금**: 처치한 영웅이 다음 런에 다른 영웅으로 등장
- **던전 테마**: 림(불/얼음/독 등)별로 다른 몬스터 풀 — **TBD**

## 8. 밸런싱 기준

- **평균 빌드 기준**: 영웅이 2~4분 사이에 죽도록 튜닝
- HP가 30초 안에 깎임 → 액티브 1~2번밖에 못 씀 → 빌드업 X (튜닝 실패)
- HP가 5:00 다 되도록 50% → 패시브 5번만 발생 → 강화 X (튜닝 실패)
- 양극단 빌드(초고속/초장기)는 도전과제/특수 영웅으로

| 4 | SpawnerHaste | P | (신규) 모든 스포너 스폰 주기 ×0.8 (영구) |
| 5 | TimeStop | A | 영웅 5s 정지 |
| 6 | Multiply → SwarmRush | A | (v0.6 효과 교체) Phantom 6마리 즉시 영웅 근처 소환 (Multiply enum 자리·SO 파일명 보존, 효과 클래스 SwarmRushEffect) |
| 7 | Slow | A | (v0.6 Debuff→Swarm + 리뉴얼) 영웅 이동속도 ×0.5 + 모든 몬스터 이동속도 ×1.3, 10s (이중 효과) |

총합 검산: 4축 × 7장 = 28장 = 패시브 16 + 액티브 12. SO 파일명·Addressable 키는 `CommonEnum.cs` 의 `ECardId` 값명과 정확히 일치 (Rule 03 §2). Multiply / Berserk 는 enum 자리만 보존하고 효과 클래스·displayName 만 교체 (각각 `SwarmRushEffect` / `GuardianRageEffect`).
