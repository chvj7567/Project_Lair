## 2. 추가 컨텐츠 후보 (권장 1개)

### Plague 스포너 배치로 패시브 2장 실효성 회복 및 둔화 빌드 활성화

- **카테고리**: 몬스터 스포너 구성 / 패시브 카드 실효성
- **요지**: 현재 Plague는 필드에 등장할 경로가 전혀 없어 SpawnPlagues·PlagueSlowBoost 패시브 2장이 완전 no-op이다. 위스프(Wisp) 스포너가 2개 중복 배치되어 있으므로 하나를 Plague 스포너로 전환하면 기존 Ring 구조(6 spawners)를 유지하면서 두 카드를 활성화하고, 컨셉 §5.2에 정의된 "둔화 + 광역 → 둔화 영웅에게 광역 데미지↑" 시너지 축을 개방할 수 있다.
- **검증/구현/시너지/데이터**: 4/2/5/3 → 종합 **16**
- **근거**:
  - `docs/design/continuous-spawn-round.md §7` — "Plague는 초기 Spawner 6개에 포함되지 않는다. … no-op이다. 이는 의도된 설계다."
  - `docs/design/continuous-spawn-round.md §3.1` — Spawner #1(0°)·#4(180°) 모두 Wisp — 2중복
  - 컨셉서 §5.2 시너지 방향성 예시: "둔화 + 광역 → 둔화 상태 영웅에게 광역 데미지↑"
  - 컨셉서 §5.1 Plague 역할 정의: "유틸 — 공격 시 영웅 둔화 20%"
  - QA 6차 리포트 §3.3 — 시뮬레이션에서 "SpiderSlowBoost(=PlagueSlowBoost)" 픽률 35%로 중위권 유지. 시뮬은 Spider(=Plague) 스포너가 있는 시나리오로 측정됨 → 실제 게임에는 해당 스포너 없음
- **MVP 범위**: 컨셉 §11.2 "몬스터 6종 ✅ / 패시브 카드 15장 ✅". Plague는 이미 §11.3 6종 목록에 포함. Spawner 구성 변경은 SpawnerConfig.asset 수치 수정 범위 내.

#### 유저 플로우 (9개 항목 모두)

1. **노출 시점·트리거**
   전투 시작 후 영웅 HP가 10% 구간을 넘을 때마다 패시브 이벤트가 발생한다. 기존에는 패시브 선택지 풀에 SpawnPlagues·PlagueSlowBoost가 뜨더라도 선택해도 아무 변화가 없어 플레이어가 무의식적으로 다른 카드를 선택해왔다. Plague Spawner 추가 후에는 두 카드가 처음으로 실질적인 트리거를 갖는다. 예를 들어 영웅 HP 70% 구간에서 SpawnPlagues 카드가 3택 중 하나로 제시되면, 선택 즉시 Plague 스포너의 동시 출력이 +1 증가한다.

2. **화면 변화**
   전투가 일시정지되고 카드 선택 팝업이 열린다. SpawnPlagues는 보라 테두리(추가 카드, 컨셉 §11.4 색상)로 표시되며, Plague Spawner가 존재하는 환경에서는 선택 후 해당 스포너 위치에서 보라색 작은 큐브(Plague 프리미티브, 스케일 0.5 납작형)가 출현하기 시작한다. PlagueSlowBoost 선택 시 Plague 의 공격 시 둔화 강도가 20%에서 최대 40%(1픽 누적 기준)로 상승하며, 영웅이 Plague에게 피격될 때마다 이동속도 감소 시각 표시(하늘색 반투명 구체)가 더 오래 유지된다.

3. **입력 행동**
   플레이어가 카드 선택 팝업에서 SpawnPlagues 또는 PlagueSlowBoost 카드를 클릭해 선택한다. 현재는 두 카드가 효과가 없으므로 사실상 "패널티 없는 낭비 픽"이었으나, Spawner 추가 후에는 "둔화 중심 빌드"를 의식적으로 구성하는 선택지가 된다. 기존에 Wisp 강화(WispHpBoost, SpawnWisps) 위주로만 돌던 패시브 픽 패턴에 Plague 축이 추가되어 전략 다양성이 증가한다.

4. **시스템 반응**
   SpawnPlagues 선택 시: `SpawnPlaguesEffect.cs` 가 `IBattleContext.GetSpawnersByMonsterType(EMonster.Plague)` 로 Plague 출력 Spawner를 조회하고, `SpawnerController.AddOutput(+1)` 을 호출한다. Plague Spawner가 존재하면 다음 스폰 사이클부터 한 주기에 2마리씩 출력된다. PlagueSlowBoost 선택 시: `PlagueSlowBoostEffect.cs` 가 Plague 글로벌 버프(SlowFactorMul ×0.75)를 등록하고, 현재 필드에 살아있는 Plague 전부에도 즉시 소급 적용된다. BaseSlowFactor 0.8 × 0.75 = 0.6으로 영웅이 Plague 피격 시 이동속도가 60%로 감속된다.

5. **반복·재발생 패턴**
   영웅이 Plague에게 피격될 때마다 10초간 이동속도 60% 감속 디버프가 재부착(갱신)된다. Plague Spawner 주기에 따라 필드에 Plague가 지속 공급되므로, 영웅은 Plague 무리와 접촉하는 동안 거의 항시 둔화 상태를 유지한다. SpawnPlagues를 중복 픽하면 Spawner 동시 출력이 +2가 되어 더 빠르게 Plague가 필드를 채운다. PlagueSlowBoost를 1픽하면 SlowFactorMul=0.75, 2픽하면 ×0.75² = 0.5625로 영웅이 이동속도 56%로 감속된다.

6. **종료·해소 조건**
   개별 둔화 디버프는 피격 후 10초가 지나면 자동 해소되고 영웅 이동속도가 원상복구된다. 영웅을 처치하면 승리로 판이 끝나고 모든 디버프가 즉시 해소된다. 5분 타임오버(패배)로 판이 종료될 때도 동일하게 즉시 해소된다. CHMPool Push 시 Plague의 OnDisable 리셋으로 다음 판에 상태 이월 없음.

7. **다른 시스템과 상호작용**
   컨셉 §5.2의 시너지 방향성("둔화 + 광역 → 둔화 상태 영웅에게 광역 데미지↑")이 활성화된다. 영웅이 Plague 둔화로 이동속도가 줄면 몬스터 무리에 더 오래 포위되어 모든 딜러(Reaper·Hex)의 DPS 기여 시간이 늘어난다. 패시브 이벤트와 액티브 이벤트가 동시에 트리거되는 큐 상황에서 Plague 카드 픽도 동일하게 일시정지 큐로 처리되므로 기존 이벤트 처리 순서에 영향 없다. Wisp 스포너가 1개에서 2개로 줄어드는 것과 반대로 Plague 스포너가 0개에서 1개가 되므로, WispHpBoost·SpawnWisps 두 카드의 필드 영향력이 소폭 줄어드는 트레이드오프가 발생한다.

8. **엣지 케이스**
   Plague 스포너가 1개뿐인 상황에서 ReplaceWispsToWraith 교체 카드가 남은 Wisp 스포너(1개)에 적용되면, 위스프 스포너 0개 · 레이스 스포너 추가 · Plague 스포너 1개 구성이 된다. SpawnWisps 카드는 Wisp 스포너가 0개이므로 no-op이 되는 2차 no-op 상황이 발생할 수 있다 — 교체 카드 연쇄 no-op은 기존에도 발생 가능한 패턴이므로 기획서 §6.3 규칙이 그대로 적용된다. Plague 스포너 추가 시 글로벌 캡(18마리)과의 관계: Plague의 Power가 2로 매우 낮아 캡 포화 속도에 미치는 영향은 미미하다(QA 시뮬에서 Spider Power=2 기준 측정값 이미 반영됨).

9. **유저 정보·피드백**
   플레이어는 선택 팝업에서 카드 설명 텍스트를 통해 효과를 확인한다. Plague 스포너가 생기면 필드에서 보라색 작은 납작 큐브가 출현하기 시작하는 시각적 변화로 픽이 즉각 반영됨을 확인할 수 있다. 영웅의 이동속도 감속은 HP 바 아래 상태 아이콘이나 하늘색 구체 이펙트(컨셉 §11.4 둔화 표시)로 피드백된다. 기존에는 SpawnPlagues를 선택해도 아무 변화가 없어 카드가 "고장난 것 아닌가?" 하는 혼란이 있었으나, Spawner 추가 후에는 즉각 시각 반응으로 선택 효과가 명확히 전달된다.

## 6. 쉬운 설명 (비개발자 요약)

현재 게임에서 '플레이그 소환' 카드와 '플레이그 강화' 카드를 고르면 화면에 아무 변화가 없다. 마치 버튼을 눌렀는데 불이 안 켜지는 상황처럼, 이 카드들은 선택해도 실제로는 아무 효과가 없다. 이유는 단순한데, 플레이그 몬스터가 나올 수 있는 출구(스포너)가 던전에 처음부터 없기 때문이다. 위스프라는 몬스터가 같은 출구를 두 개 독점하고 있는 구조다. 그래서 이번에 제안하는 것은: 위스프 출구 하나를 플레이그 출구로 바꿔서, 잠들어 있던 패시브 카드 2장이 실제로 작동하게 만들고 "느리게 만들기" 전략 루트를 열어주는 것이다.
